# Система анализа популярности фотографий

---

# Общая концепция

Система предназначена для сбора, хранения и анализа статистики фотографий с внешнего сайта.

Основная цель системы:
- отслеживание популярности фотографий
- анализ динамики просмотров/лайков/комментариев
- выявление трендов
- предоставление удобного интерфейса аналитики

Система состоит из трех основных частей:

1. Сбор данных с внешнего сайта
2. Хранение и обработка данных
3. Веб-интерфейс аналитики

---

# Основные сценарии использования

## Анализ отдельной фотографии

Пользователь может:

- просматривать:
    - количество просмотров
    - лайков
    - комментариев
- анализировать динамику изменений
- смотреть графики роста популярности
- сравнивать периоды активности

---

## Анализ всей коллекции

Пользователь может:

- видеть:
    - самые популярные фотографии
    - самые быстрорастущие фотографии
    - фотографии с лучшим engagement
- анализировать:
    - динамику аккаунта
    - активность по периодам
    - распределение популярности

---

# Источники данных

## Внешний сайт с фотографиями

Из внешнего сайта извлекаются:

### Метаданные фотографии

- идентификатор фото
- URL фотографии
- дата публикации
- название
- описание
- автор
- теги
- категории / альбомы
- EXIF данные
- технические параметры

---

### Динамическая статистика

- просмотры
- лайки
- комментарии
- избранное (если доступно)
- рейтинг (если доступно)

---

# Архитектура хранения данных

## 1. Хранилище изображений

Система должна поддерживать:

### Возможные варианты хранения

- хранение только URL
- локальное хранение
- cloud object storage

---

### Дополнительно

Отдельное хранение:
- thumbnails
- preview изображений

Это необходимо для:
- ускорения SPA
- снижения нагрузки
- быстрого отображения галереи

---

## 2. Хранение метаданных

Метаданные хранятся отдельно от статистики.

### Причины

Метаданные:
- изменяются редко
- имеют реляционную структуру
- часто используются для фильтрации и поиска
